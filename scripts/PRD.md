# 🧠 IdeaStorm PRD v1.1
**AI Multi-Agent Brainstorming Tool**

> 작성일: 2026.03 · 버전: v1.1
> 변경사항: 사용자 역할 재정의(진행자→참가자) / Facilitator 에이전트 추가 / 3라운드 수렴 메커니즘 / 페르소나 셋 구조로 전환

---

## 1. 프로젝트 개요

### 한 줄 정의
> CBL(Challenge Based Learning) 프레임워크를 기반으로, Facilitator 에이전트가 세션을 진행하고 여러 페르소나 에이전트들과 사용자가 함께 참가자로서 브레인스토밍을 수행하는 대화형 아이디에이션 툴.

### 1.1 배경 및 문제 정의

해커톤, 공모전, 사이드 프로젝트 기획 등 아이디에이션 상황에서 **혼자 CBL 프레임워크를 수행할 때** 다음 세 가지 문제가 발생한다.

| 문제 | 원인 | 영향 |
|------|------|------|
| 아이디어 편향 | 자신이 편한 관점에서만 사고 | 탐색 범주가 좁아짐 |
| 평가 불안 | 스스로 아이디어를 검열 | 대담한 아이디어 차단 |
| 탐색 중단 | 다음 단계로 어떻게 가야 할지 모름 | 프레임워크가 중간에 끊김 |

### 1.2 솔루션 컨셉

**Facilitator 에이전트**가 세션 전체를 진행하고, **페르소나 에이전트들**과 **사용자**가 동등한 참가자로 함께 아이디어를 발산·수렴한다. 사용자는 더 이상 진행자가 아니라 "또 다른 참가자"로서 자신의 관점을 자유롭게 던질 수 있다.

> **핵심 원칙**: "AI가 아이디어를 대신 내주는 게 아니라, 사용자가 스스로 생각하도록 질문으로 자극한다." — CHI 2025 연구 기반 설계 원칙

## 2. 참가자 구조

### 2.1 역할 정의

| 역할 | 주체 | 하는 것 |
|------|------|---------|
| **Facilitator** | AI 에이전트 (고정) | 세션 진행, 단계 전환, 수렴 주도, 충돌 중재, 전체 요약 |
| **페르소나 에이전트** | AI 에이전트 (셋 선택) | 각자 페르소나 관점에서 아이디어·질문 생성, 서로 반응 |
| **사용자** | 인간 | 페르소나 에이전트들과 동등한 참가자, 자유롭게 의견 제시 |

### 2.2 Facilitator 에이전트

세션의 흐름을 관리하는 **중립적 진행자**. 아이디어를 직접 생성하지 않으며, 오직 프로세스에만 집중한다.

**하는 것**
- CBL 단계 전환 안내 ("이제 수렴할 시간이에요")
- 조용한 참가자 독려 ("사용자님은 어떻게 생각하세요?")
- 에이전트 충돌 중재 및 요약
- 발산된 아이디어를 2~3개 클러스터로 압축
- 수렴 투표 진행

**하지 않는 것**
- 직접 아이디어 제안
- 특정 방향으로 유도
- 사용자 의견 평가

## 4. 전체 사용자 플로우

CBL 4단계를 Facilitator가 진행하고, 페르소나 에이전트들과 사용자가 함께 참가한다.
각 단계는 **발산 → 반응 → 수렴**의 3라운드 구조를 반복한다.

```
페르소나 셋 선택
        ↓
[PHASE 1] Big Idea 설정
        ↓
[PHASE 2] Essential Questions 도출
        ↓
[PHASE 3] Challenge (HMW) 확정
        ↓
[PHASE 4] 포스트잇 보드
        ↓
최종 리포트 생성
```

---

### PHASE 1 · Big Idea 설정

**발산** — 각 에이전트가 자신의 관점에서 키워드 3~5개씩 생성. 사용자도 자유롭게 키워드 제시.

**반응** — 에이전트들이 서로의 키워드에 한 마디씩 반응 (공감 / 확장 / 반박). Facilitator가 전체 흐름 요약.

**수렴** — Facilitator가 2~3개 후보로 압축 → 전원 투표 → 과반 득표 항목 선정. 과반 미달 시 상위 2개 결선 투표. 사용자 직접 지정 가능 **(Override)**.

---

### PHASE 2 · Essential Questions

**발산** — 각 에이전트가 Big Idea 관련 질문 3~5개씩 생성. 사용자도 직접 질문 추가.

**반응** — 에이전트들이 서로의 질문에 반응해 더 날카로운 질문으로 발전. Facilitator가 유사 질문 묶기.

**수렴** — Facilitator가 핵심 후보 3~5개로 압축 → 전원 투표 → 상위 3개 선정.

---

### PHASE 3 · Challenge 도출

**발산** — 각 에이전트가 HMW 초안 1~2개씩 제안. 사용자도 직접 HMW 문장 제안 가능.

**반응** — 에이전트들이 서로의 HMW에 반응해 개선 제안. Facilitator가 겹치는 방향 병합.

**수렴** — Facilitator가 최종 후보 2~3개 제시 → 전원 투표 → 1개 확정. 사용자 직접 수정 가능 **(Override)**.

---

### PHASE 4 · 포스트잇 보드

**발산** — 각 에이전트가 자기 페르소나 관점에서 포스트잇 자유 생성. 사용자도 포스트잇 직접 추가. 사용자가 특정 에이전트 포스트잇에 답하면 → 에이전트가 다음 질문 동적 생성 **(대화형 심문)**.

**반응** — 에이전트들이 서로의 포스트잇에 반응 포스트잇 추가. Facilitator가 미탐색 영역 알림. AI 실시간 클러스터링.

**수렴** — Facilitator가 클러스터 기반으로 핵심 방향 2~3개 요약 → 전원 투표 → 최종 방향 선정 → 리포트 생성.

## 5. 3라운드 수렴 메커니즘

모든 단계에서 동일한 3라운드 구조를 반복한다.

```
[Round 1] 발산
  모든 참가자(에이전트 + 사용자)가 자유롭게 의견 제시
  Facilitator: 전체 내용 요약 정리
        ↓
[Round 2] 반응
  각자가 타인 의견에 한 마디씩 반응 (동의 / 반박 / 결합)
  자연스럽게 클러스터가 형성됨
  Facilitator: 유사 의견 묶기, 클러스터 라벨링
        ↓
[Round 3] 수렴 투표
  Facilitator가 2~3개 후보로 압축 제시
  전원 1표씩 투표
  과반 득표 항목 선정
  과반 미달 시 → 상위 2개 결선 투표
```

### 투표 구조

| 투표 주체 | 투표 수 | 기준 |
|---------|--------|------|
| 각 페르소나 에이전트 | 1표 | 자신의 페르소나 관점에서 가장 적합한 항목 |
| 사용자 | 1표 | 자유 |
| Facilitator | 0표 | 투표하지 않음 (중립 유지) |

> **Override**: 사용자는 매 단계에서 투표 결과와 무관하게 직접 방향을 결정할 수 있다. Override 시 Facilitator가 "다른 참가자들의 의견 요약"을 한 번 더 보여준다.

## 6. 에이전트 간 상호작용

### 6.1 반응 유형

| 반응 유형 | 예시 | 효과 |
|---------|------|------|
| 동의 + 확장 | 몽상가의 "AI 친구" 아이디어를 현실주의자가 구체적 구현 방식으로 발전 | 아이디어 깊이 증가 |
| 반박 | 비평가가 비저너리 CEO의 시장 가정에 반례 제시 | 현실성 검증 |
| 결합 | Facilitator가 두 에이전트의 아이디어를 하나로 합치기 제안 | 교차 융합 |
| 질문 심화 | 사용자 답변을 받아 에이전트가 더 깊은 질문 동적 생성 | 탐색 깊이 증가 |

### 6.2 충돌 상황

에이전트들이 서로 반대 방향을 가리킬 때, Facilitator가 **"충돌 감지"** 를 선언하고 양측 주장을 요약한 뒤 사용자에게 방향 선택을 위임한다.

## 7. 핵심 기능 정의

### 7.1 포스트잇 보드
- 에이전트별 색상으로 구분된 포스트잇 실시간 생성
- 드래그로 직접 배치 가능
- AI 자동 클러스터링 + 라벨링
- 미탐색 영역 알림
- 최종 보드 이미지 내보내기

### 7.2 대화 패널
- 전체 참가자 동시 대화 뷰 (메인)
- 특정 에이전트와 1:1 채팅 가능
- 사용자 답변 → 에이전트 동적 질문 생성

### 7.3 Facilitator 패널
- 현재 단계 및 진행 상황 요약
- 클러스터링 결과 시각화
- 수렴 투표 UI

### 7.4 최종 리포트
- 확정된 Big Idea / Challenge 문장
- 포스트잇 요약 + 클러스터 인사이트
- "가장 많이 탐색된 관점" vs "가장 덜 탐색된 관점" 시각화
- 마크다운 / PDF 내보내기

## 8. 연구 기반 설계 근거

| 해결하는 문제 | 연구 근거 | 설계 해법 |
|-------------|---------|---------|
| 생산 차단 | Diehl & Stroebe (1987) | 에이전트 병렬 생성 — 동시에 모든 페르소나에서 출력 |
| 평가 불안 | Camacho & Paulus (1995) | 에이전트는 판단하지 않고 자기 관점만 제시 |
| 인지적 고착 | Kohn & Smith (2011) | 미탐색 영역 알림 + 다른 페르소나 자극 |
| AI 소유감 감소 | CHI 2025 연구 | 사용자도 동등한 참가자, 에이전트와 같은 위치 |
| 하향 성과 매칭 | Paulus & Dzindolet (1993) | 에이전트 충돌 + Facilitator 중재로 상향 자극 |
| AI 조기 개입 | CHI 2025 "Timing Matters" | Round 1은 자유 발산, 수렴은 Round 3에만 |
| 아이디어 선별 품질 | Rietzschel et al. (2006) | 3라운드 구조로 생성과 선별을 명확히 분리 |

## 9. 경쟁 차별화

| 비교 대상 | 핵심 차이 |
|---------|---------|
| ChatGPT / Claude | 단일 AI가 아이디어 생성 → 복수 페르소나가 서로 충돌하며 탐색 |
| Miro / FigJam | 보드 도구 + 보조 AI → Facilitator가 세션 전체를 주도적으로 진행 |
| Notion AI | 문서 작성 보조 → CBL 전체 플로우 구조화 + 수렴 프로세스 |
| 혼자 하는 CBL | 사용자가 진행자 + 참가자 동시 → 사용자는 참가에만 집중 |

### 12.2 화면 구성 요소

**인피니트 캔버스 (우측 메인)**
- 무한 스크롤 + 핀치 줌
- 에이전트별 고유 색상 포스트잇 (자동 배치 → 드래그로 재배치 가능)
- 포스트잇 클릭 시 상세 내용 + 반응 스레드 펼치기
- AI 클러스터링 시 점선 테두리로 그룹 표시
- 미탐색 영역은 회색 점선 영역으로 시각화

**대화 패널 (좌측)**
- Facilitator 발언은 상단 고정 + 강조
- 에이전트별 아바타 + 색상으로 구분
- 사용자 입력창: 텍스트 입력 or 포스트잇으로 바로 보드에 추가 선택
- 1:1 탭 전환으로 특정 에이전트 집중 대화 가능

**상단 툴바**
- 현재 Phase + 진행률 바
- 참가 중인 에이전트 아바타 목록
- 세션 설정 (페르소나 셋 변경, AI 모델 선택)

**하단 플로팅 액션바**
- `[투표 시작]` — Round 3 진입 시 활성화
- `[다음 단계 →]` — 수렴 완료 후 활성화
- `[Override]` — 항상 표시, 사용자 직접 결정

### 12.3 포스트잇 상세 스펙

### 14.2 Step 1 — AI 모델 연결

서비스 자체 계정 없음. **AI 모델 연결이 곧 로그인**이다.


## 15. 에이전트 발언 순서 및 타이밍

### 15.1 발언 순서 — 순차적 구조

사용자가 흐름을 따라갈 수 있도록 **순차적** 으로 발언한다.

```
Round 1 발언 순서 (예: 셋 B 스타트업)

  🎯 Facilitator   → Round 시작 선언 + 질문 제시
        ↓
  🟡 에이전트 1    → 발언
        ↓
  🔴 에이전트 2    → 발언 (앞 발언 참조 가능)
        ↓
  🔵 에이전트 3    → 발언
        ↓
  🟢 에이전트 4    → 발언
        ↓
  👤 사용자        → 발언 (선택, 패스 가능)
        ↓
  🎯 Facilitator   → 전체 요약 → Round 2 전환 선언
```

**에이전트 발언 간격**: 이전 발언 완료 후 **1.5초 딜레이** → 다음 에이전트 발언 시작  
(너무 빠르면 읽기 어렵고, 너무 느리면 답답함)

### 15.2 Round 타이밍 — 침묵 감지 기반

```
마지막 발언 (에이전트 or 사용자) 완료
        ↓
   ⏱️ 15초 침묵 감지
        ↓
  🎯 Facilitator: "모두 이야기 나눴나요?
                   다음 라운드로 넘어갈까요?"
        ↓
   ┌─────────────────────────────┐
   │  [넘어가기]   [더 이야기하기] │
   └─────────────────────────────┘
        ↓
  사용자가 응답하지 않으면 → 30초 후 자동 전환
```

**라운드별 침묵 감지 기준**

| 라운드 | 침묵 감지 시간 | 자동 전환 |
|--------|-------------|---------|
| Round 1 (발산) | 15초 | 30초 후 자동 |
| Round 2 (반응) | 10초 | 20초 후 자동 |
| Round 3 (투표) | 투표 완료 즉시 | — |

### 15.3 사용자 발언 타이밍

- 에이전트 순번이 끝난 후 **사용자 차례** 표시
- 사용자는 **패스** 가능 (강제 발언 없음)
- 사용자가 에이전트 발언 중 끼어들고 싶으면 → `[지금 말하기]` 버튼으로 인터럽트 가능
- 인터럽트 시 현재 발언 중인 에이전트는 발언 완료 후 사용자에게 차례 넘김

### 15.4 에이전트 발언 길이 제한

무한 발언 방지를 위한 토큰 제한.

| 단계 | 최대 발언 길이 |
|------|-------------|
| Round 1 발산 | 150자 이내 |
| Round 2 반응 | 100자 이내 |
| Round 3 투표 이유 | 50자 이내 |
| Facilitator 요약 | 200자 이내 |

## 16. Phase별 캔버스 사용 방식

모든 Phase가 **하나의 인피니트 캔버스** 위에서 진행된다. Phase가 바뀌어도 이전 포스트잇은 사라지지 않고 캔버스에 누적된다.

### 16.1 캔버스 구역 구성

```
┌──────────────────────────────────────────────────────────────┐
│                    인피니트 캔버스                            │
│                                                              │
│  [ PHASE 1 구역 ]     [ PHASE 2 구역 ]     [ PHASE 3 구역 ] │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐  │
│  │ Big Idea    │  →   │ Questions   │  →   │  Challenge  │  │
│  │ 키워드 포스잇│      │ 질문 포스잇 │      │  HMW 포스잇 │  │
│  └─────────────┘      └─────────────┘      └─────────────┘  │
│                                                              │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                  PHASE 4 구역                          │  │
│  │              포스트잇 자유 배치 보드                    │  │
│  └───────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

### 16.2 Phase별 포스트잇 형태

| Phase | 포스트잇 내용 | 크기 | 배치 방식 |
|-------|------------|------|---------|
| Phase 1 | 키워드 (1~3단어) | Small 80×80 | 에이전트별 행으로 자동 정렬 |
| Phase 2 | 질문 문장 | Medium 160×80 | 에이전트별 행으로 자동 정렬 |
| Phase 3 | HMW 문장 | Medium 200×80 | 에이전트별 행으로 자동 정렬 |
| Phase 4 | 자유 아이디어 | Large 160×160 | 자유 배치 (드래그 가능) |

### 16.3 선정된 항목 표시

투표로 선정된 포스트잇은 **골드 테두리 + 핀 아이콘**으로 강조되고, 다음 Phase 구역으로 자동 복사된다.

```
선정 전                선정 후
┌──────────┐          ┌──────────┐ 📌
│자취생 식비│    →     │자취생 식비│ ← 골드 테두리
└──────────┘          └──────────┘
                            ↓
                       Phase 2 구역 상단에 고정 표시
```

### 16.4 캔버스 이동 및 뷰

| 기능 | 동작 |
|------|------|
| 현재 Phase 포커스 | 단계 전환 시 해당 구역으로 자동 스크롤 |
| 전체 보기 | `[📐 맞춤]` 버튼으로 전체 캔버스 한눈에 |
| Phase 이동 | 상단 Phase 탭 클릭으로 해당 구역 이동 |
| 이전 Phase 참조 | 항상 캔버스에 남아있어 드래그로 자유 이동 가능 |
